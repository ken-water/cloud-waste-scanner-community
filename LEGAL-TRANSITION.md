# Legal Transition (Community Repository)

## Purpose

This file records the transition from historical Pro-era repository wording to Community repository legal posture.

## What Changed

- `README.md` is now Community-focused.
- `LICENSE` is now a non-commercial software license (PolyForm Noncommercial 1.0.0 style terms).
- Community branch/runtime positioning is local-first and non-hosted for core workflows.

## Historical Versions and Tags

Git history is immutable unless force-rewritten. Therefore:

- Older tags/commits may still display legacy Pro wording or older legal context.
- Those historical snapshots are retained for auditability and release traceability.
- The current `main` branch documents the active Community legal baseline.

## Interpretation Rule

For current Community usage, interpret licensing and repository intent according to:

1. `LICENSE` on current `main`
2. `README.md` on current `main`
3. This transition file

in that order.

## Commercial Use

Commercial usage is outside the Community license grant and requires a separate commercial agreement.
