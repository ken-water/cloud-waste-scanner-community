# Cloud Waste Scanner Community v3.0.0

Community `v3.0.0` resets the release line for the open client-first roadmap.

This release establishes a clearer Community / Team / Enterprise boundary and introduces the first Community skill family for local evidence explanation, weekly review, and action preparation.

This release-line reset does **not** imply that every unchanged desktop or SDK binary artifact was rebuilt as `3.0.0`.

## Highlights

- Community release line reset to `3.0.0`
- clearer Community / Team / Enterprise boundary
- first Community skill: `cws-report-explainer`
- second Community skill: `cws-weekly-brief`
- third Community skill: `cws-playbook-writer`
- formal Community skill strategy, runtime modes, and monetization guardrails
- conservative Team skill readiness and closure planning

## Included

- Community product-boundary clarification
- Community skills strategy

- centralized identity or audit rollout
- mandatory binary rebuild for unchanged desktop artifacts

## Why this release exists

The previous `2.9.x` line mixed historical Pro-era assumptions with an evolving Community scope.

`v3.0.0` resets the Community story around a cleaner rule:

- Community: local-first discovery and evidence
- Team: governance execution
- Enterprise: centralized identity, audit, and control

## What is new

### `cws-report-explainer`

Turn CWS evidence into:

- operator explanation
- finance summary
- executive brief
- weekly action list

Supports:

- CWS local API
- exported JSON / CSV / TXT evidence

### `cws-weekly-brief`

Turn current and prior evidence into:

- weekly scan summary
- current-vs-prior comparison
- repeated waste signal highlighting
- founder/operator weekly checkpoint

Supports:

- CWS local API
- exported current/prior evidence

### `cws-playbook-writer`

Turn findings into:

- pre-check
- execution suggestion
- verification step
- rollback caution

Supports:

- local API findings
- exported findings
- normalized evidence bundles

### Community skill rules

This release makes the following rules explicit:

- Community skills explain and prepare
- Team skills coordinate and execute
- Enterprise skills control and audit
- Community skills support both:
  - `connected-mode`
  - `file-only-mode`

## Upgrade guidance

If you are using Community today:

1. keep using the local-first desktop workflow
2. use `cws-report-explainer` for role-specific summaries
3. use `cws-weekly-brief` for current-vs-prior weekly review
4. use `cws-playbook-writer` for safer evidence-to-action preparation
5. treat Team and Enterprise skill ideas as roadmap, not current GA surface
