# Weekly Brief Inputs

## Preferred data

Use the local API when available:

- `GET /v1/scans`
- `GET /v1/findings`
- `GET /v1/reports`

## File fallback

Accept:

- current findings export
- current report export
- prior findings or report export
- pasted review notes

## Required minimum

Minimum viable weekly brief:

- one current evidence set

Preferred weekly brief:

- current evidence set
- prior evidence set for comparison

## Interpretation rules

- If prior data is missing, say comparison is limited.
- If repeated assets cannot be matched exactly, still compare by provider and resource-type concentration.
- Treat projected waste as estimated impact, not booked savings.
